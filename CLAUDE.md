# 루멘수학 (Lumen Math) 프로젝트

한국 수학학원 "루멘수학"의 학원 운영 시스템. 원장(김정수 선생님)이 Claude와 함께 개발 중.
개발자가 아닌 원장님이 운영하는 프로젝트이므로 **설명은 한국어로, 비개발자도 이해할 수 있게** 할 것.

## 구성 요소

| 파일 | 역할 |
|------|------|
| `lumen_v15-1.html` | **학원앱(선생님용)** — 최신 버전. 단일 HTML 파일(약 3.5만 줄)에 CSS/JS 포함 |
| `student_v2-2.html` | **학생앱** — 최신 버전. 단일 HTML 파일 |
| `lumen_v10~14-*.html`, `student_v1-*.html` | 이전 버전 히스토리 (참고용, 수정 금지) |
| `lumen_PRD_v1.docx` | 기획서 |
| `lumen_dashboard_v1-5.jsx` | 대시보드 프로토타입 |

**버전 규칙**: 파일을 직접 수정하지 않고 새 버전 파일을 만드는 방식으로 발전해 왔음
(예: `lumen_v15-1.html` → `lumen_v15-2.html`). 큰 수정 시 이 규칙을 따를 것.

## 데이터 구조 (두 앱의 연결 고리)

두 앱은 **Supabase** (`https://bhkkkbcytcrlxhrtjgen.supabase.co`)를 공유한다.

- **`lumen_store` 테이블** — key/value(jsonb) 범용 저장소. 핵심 채널.
  - 학원앱이 localStorage 데이터를 이곳에 동기화(union-merge, 삭제 전파 안 함)
  - 주요 키: `or_studentdb`(학생 등록부), `or_books`(교재), `submissions_<학생코드>`(제출물),
    `student_planner_<code>`(플래너 점수), `bookprog_<code>`(교재 진도), `lumen_announcements`(공지)
- **`aha_notes` 테이블** — 학생이 올리는 질문/오답 기록 (사진+교재+페이지+문항번호+틀린이유 태그, `status: pending` → 선생님이 해결 처리)
- **`announcement_reads`**, **`vod_watch_logs`** — 공지 읽음, VOD 시청 로그
- **Storage 버킷**: `photos`(플래너/풀이 사진), `recordings`(녹음), `school_pdfs`, `aha_photos`
- 학생 식별자: 6자리 `lumen_rec_code` (학생앱 로그인도 이 코드로)

학원앱 자체 데이터는 localStorage 중심 (`or_studentdb`, `or_books`, `lumen_type_db`(유형DB),
`school_scores`, `mock_exams`, `lumen_attendance` 등). AI 키는 `or_api_key`/`or_gemini_key`.

## 주요 기능 현황

**학원앱**: 학생관리, 출결·보충, 교재·유형 관리, 플래너(사진 AI 분석), 숙제 4단계 스테이지
(풀이→채점→오답→아하노트, 단계별 XP), 레벨/뱃지 게이미피케이션, 아하노트 관리, VOD 강의실,
기출모의/학교성적/자체시험, AI 일괄 피드백, **매쓰플랫 월간 리포트(PDF 업로드 → AI 파싱)**

**학생앱**: 하브루타 녹음 제출, 플래너 사진 제출(EXIF 검증, 스트릭🔥), 플래너 점수/랭킹,
아하노트(질문+오답 기록), 로드맵(교재 진도), 공지사항

## 🎯 진행 중: 매쓰플랫 정오답 자동 연동 (핵심 프로젝트)

**목표**: 매쓰플랫(mathflat.com, 프리윌린)에서 학생별·교재별 문항 단위 정오답 데이터를
매일 자동 수집 → Supabase 저장 → 학원앱/학생앱에서 활용.

현재 매쓰플랫 연동은 "월간 리포트 PDF를 수동 업로드"하는 수준(`lumen_mf_reports`).
이를 **문항 단위 자동 수집**으로 발전시키는 것이 목표.

### 방식
- 매쓰플랫은 공개 API가 없음 → **Playwright 브라우저 자동화**로 선생님용 웹
  (teacher.mathflat.com)에 로그인하여 수집 (또는 로그인 후 내부 API 호출 관찰·활용)
- 계정은 환경변수 `MATHFLAT_ID`, `MATHFLAT_PASSWORD`에 저장됨 (클라우드 환경 설정에 등록,
  **절대 코드/저장소에 비밀번호를 넣지 말 것**)
- 주의: 매쓰플랫은 동시 로그인 시 기존 접속이 끊길 수 있음 → 자동 수집은 새벽 시간대 권장
- 매일 자동 실행: 예약 실행(Routine/cron) 사용 예정

### 단계별 계획
1. **로그인 테스트** — Playwright로 매쓰플랫 로그인 성공 확인 ← 다음 할 일
2. **수집기 개발** — 학생 목록 → 교재별 페이지별 문항 정오답(O/X)과 채점 시각 수집
3. **저장 설계** — Supabase에 문항 단위 정오답 테이블 신설
   (예: `mf_answer_records`: 학생코드, 교재, 페이지, 문항번호, 정오, 채점일시, 단원/유형)
4. **앱 기능 구현** (아래 로드맵)
5. **매일 자동 실행** 예약 + 실패 알림

### 활용 기능 로드맵
- **에빙하우스 망각곡선 복습 스케줄러**: 틀린 문항마다 1→3→7→14→30일 주기로 복습 시점
  자동 계산. 복습에서 또 틀리면 주기 리셋. 학생앱에 "오늘의 복습" 카드, 학원앱에 학생별 복습 큐
- **아하노트 ↔ 정오답 연동**: 아하노트에 이미 교재+페이지+문항번호가 있으므로
  `mf_answer_records`와 매칭 → 질문한 문제의 실제 정오/재복습 결과 추적
- **취약 유형 분석**: 기존 유형DB(`lumen_type_db`)·유형성취도 화면과 결합, 단원·유형별 정답률 추이
- **수업 전 브리핑**: 오늘 수업 학생의 최근 오답·취약점 요약
- **위험 신호 감지**: 정답률 하락 추세, 채점 기록 끊긴 학생 알림
- **학부모 리포트 자동화**: 기존 매쓰플랫 PDF 리포트를 자동 수집 데이터로 대체/보강
- **XP 연동**: 복습 완료 시 기존 레벨/XP 시스템에 반영

## 작업 규칙

- 비밀번호·API 키는 절대 커밋하지 않는다 (환경변수만 사용)
- 학생 개인정보(이름·성적)가 포함된 실데이터를 저장소에 커밋하지 않는다
- 기존 버전 파일들은 수정/삭제하지 않는다 (히스토리)
- 앱은 단일 HTML 파일 구조를 유지한다 (원장님이 파일 하나로 배포하는 방식 선호)
- 새 스크립트(수집기 등)는 `sync/` 폴더에 만든다
- 문서는 `docs/` 폴더에 둔다
